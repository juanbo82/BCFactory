¿Por qué son importantes los Permission Sets?

Cada extensión que publicas en Business Central necesita definir qué permisos necesita para funcionar correctamente. Si no defines Permission Sets adecuados, tus usuarios recibirán errores de permisos al intentar usar tu funcionalidad.

Formato AL (recomendado)

Desde BC 20+, el formato recomendado es AL en lugar de XML:

permissionset 50100 "MYAPP Full Access"
{
    Caption = 'My App - Full Access';
    Assignable = true;

    Permissions =
        tabledata "My Custom Table" = RIMD,
        table "My Custom Table" = X,
        page "My Custom List" = X,
        page "My Custom Card" = X,
        codeunit "My Custom Logic" = X,
        report "My Custom Report" = X;
}

Niveles de permisos

  • R - Read (lectura)
  • I - Insert (inserción)
  • M - Modify (modificación)
  • D - Delete (eliminación)
  • X - Execute (ejecución, para objetos no-tabla)

Permission Set Extension

Si necesitas añadir permisos a un permission set existente (por ejemplo, el de BC base):

permissionsetextension 50100 "MYAPP D365 Basic Ext" extends "D365 BASIC"
{
    Permissions =
        tabledata "My Custom Table" = R,
        page "My Custom List" = X;
}

Errores comunes

  • Olvidar tabledata: Los permisos de table y tabledata son diferentes. table es para el esquema, tabledata para los datos
  • No incluir permisos indirectos: Si tu codeunit lee de tabla X, los usuarios necesitan permisos de lectura en tabla X
  • Assignable = false por defecto: Si no pones Assignable = true, el administrador no podrá asignar el permission set a usuarios
  • No crear un permission set de solo lectura: Siempre ofrece al menos dos niveles: completo y solo lectura

Buenas prácticas

  • Crea al menos dos permission sets: uno de acceso completo y otro de solo lectura
  • Usa prefijos consistentes en los nombres: MYAPP Full Access, MYAPP Read Only
  • Documenta qué hace cada permission set en el Caption
  • Testea siempre con un usuario que tenga solo tu permission set asignado
  • Revisa los permisos indirectos: si tu código lee Setup tables, incluye esos permisos
