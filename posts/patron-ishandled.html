¿Qué es el patrón IsHandled?

El patrón IsHandled es una convención de diseño en Business Central que permite a otros desarrolladores interceptar y reemplazar la lógica de negocio sin modificar el código original. Es la base de la extensibilidad en BC moderno.

Cuando publicas un evento con un parámetro var IsHandled: Boolean, estás diciendo: "Antes de ejecutar mi lógica, dejo que alguien más la maneje por completo si lo necesita".

Implementación básica

Aquí tienes la estructura fundamental del patrón:

procedure CalculateDiscount(var SalesLine: Record "Sales Line")
var
    IsHandled: Boolean;
begin
    IsHandled := false;
    OnBeforeCalculateDiscount(SalesLine, IsHandled);
    if IsHandled then
        exit;

    // Lógica por defecto del descuento
    if SalesLine.Quantity > 100 then
        SalesLine."Line Discount %" := 10;

    OnAfterCalculateDiscount(SalesLine);
end;

[IntegrationEvent(false, false)]
local procedure OnBeforeCalculateDiscount(
    var SalesLine: Record "Sales Line";
    var IsHandled: Boolean)
begin
end;

[IntegrationEvent(false, false)]
local procedure OnAfterCalculateDiscount(
    var SalesLine: Record "Sales Line")
begin
end;

Cómo suscribirse al evento

Desde otra extensión, un desarrollador puede reemplazar completamente la lógica de descuento:

[EventSubscriber(ObjectType::Codeunit, Codeunit::"My Discount Logic",
    'OnBeforeCalculateDiscount', '', false, false)]
local procedure HandleCustomDiscount(
    var SalesLine: Record "Sales Line";
    var IsHandled: Boolean)
begin
    if SalesLine."Customer Price Group" = 'VIP' then begin
        SalesLine."Line Discount %" := 25;
        IsHandled := true;
    end;
end;

Reglas importantes

  • Siempre inicializa IsHandled := false antes del evento
  • Comprueba IsHandled inmediatamente después de invocar el evento
  • Pasa siempre como var el parámetro IsHandled para que los suscriptores puedan modificarlo
  • No pongas lógica adicional después del if IsHandled then exit y antes de la lógica por defecto
  • Usa OnBefore para eventos que permiten cancelar la lógica y OnAfter para eventos que extienden la lógica

¿Cuándo NO usar IsHandled?

No todos los eventos necesitan el patrón IsHandled. Úsalo solo cuando la lógica que publicas puede ser completamente reemplazada. Si solo quieres notificar que algo ha ocurrido, un evento sin IsHandled es suficiente.

Este patrón es utilizado extensivamente por Microsoft en el código base de Business Central, especialmente en codeunits de posting como Sales-Post y Purch.-Post.
