Introducción

Las APIs REST de Business Central permiten que aplicaciones externas lean y escriban datos directamente en tu entorno BC. Combinándolas con Power Automate, puedes automatizar procesos sin escribir una sola línea de código AL.

Configuración de la autenticación OAuth 2.0

Para conectar Power Automate con BC vía API, necesitas registrar una aplicación en Microsoft Entra ID (Azure AD):

  1. Ve al portal de Azure > Microsoft Entra ID > App registrations
  2. Crea un New registration con el nombre de tu aplicación
  3. Añade un Client Secret en Certificates & secrets
  4. En API permissions, añade Dynamics 365 Business Central > API.ReadWrite.All
  5. Concede Admin consent

Estructura de las URLs de la API

Las APIs de BC siguen esta estructura base:

https://api.businesscentral.dynamics.com/v2.0/{tenantId}/{environmentName}/api/v2.0/

Algunos endpoints comunes:

// Clientes
GET /customers
GET /customers({id})

// Pedidos de venta
GET /salesOrders
POST /salesOrders

// Artículos
GET /items
PATCH /items({id})

Ejemplo en Power Automate

Para crear un flujo que lea clientes de BC:

  1. Añade el trigger que necesites (manual, schedule, etc.)
  2. Añade una acción HTTP
  3. Configura el método GET y la URL del endpoint
  4. En Authentication, selecciona Active Directory OAuth
  5. Rellena Tenant, Client ID y Client Secret
  6. Resource: https://api.businesscentral.dynamics.com

Ejemplo de POST (crear cliente)

{
    "number": "C00100",
    "displayName": "Contoso Ltd.",
    "type": "Company",
    "email": "info@contoso.com",
    "phoneNumber": "+34 600 123 456"
}

Consejos

  • Usa $filter para filtrar resultados: ?$filter=displayName eq 'Contoso'
  • Usa $select para pedir solo los campos que necesitas
  • Usa $top para limitar resultados y paginar
  • Siempre incluye el header If-Match con el @odata.etag al hacer PATCH
  • Las APIs custom se definen como page con PageType = API en AL
