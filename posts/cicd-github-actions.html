¿Por qué CI/CD para Business Central?

Implementar CI/CD (Integración y Despliegue Continuo) en tus proyectos de Business Central te permite detectar errores antes de llegar a producción, automatizar la compilación y testing de tus extensiones, y acelerar el ciclo de desarrollo.

Prerrequisitos

  • Repositorio en GitHub
  • Conocimientos básicos de Docker (para contenedores BC)
  • Acceso a Business Central Artifacts
  • Familiaridad con YAML

Estructura del workflow

Crea un archivo .github/workflows/ci.yml en tu repositorio:

name: CI - Build and Test

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4

      - name: Build Extension
        uses: microsoft/AL-Go-Actions/Build@main
        with:
          shell: powershell
          parentTelemetryScopeJson: '{}'

      - name: Run Tests
        uses: microsoft/AL-Go-Actions/RunTests@main
        with:
          shell: powershell

Usando AL-Go for GitHub

AL-Go for GitHub es la herramienta oficial de Microsoft para CI/CD con Business Central. Simplifica enormemente la configuración:

  1. Usa el template de AL-Go-PTE para per-tenant extensions
  2. Configura los settings en .AL-Go/settings.json
  3. Los workflows se generan automáticamente
{
    "country": "es",
    "repoVersion": "1.0",
    "artifact": "bcartifacts/sandbox//es/latest",
    "testFolders": ["test/"]
}

Flujo recomendado

  1. Push a develop → Build automático + tests
  2. Pull Request a main → Build + tests + code review
  3. Merge a main → Build + tests + generación del .app
  4. Release tag → Publicación automática del artefacto

Consejos

  • Usa caching de artefactos para acelerar los builds
  • Separa los tests unitarios de los tests de integración
  • Configura branch protection rules para requerir que el CI pase antes de merge
  • Guarda los secrets (passwords, tokens) en GitHub Secrets, nunca en el código
  • Usa AL-Go siempre que sea posible; es mantenido por Microsoft
